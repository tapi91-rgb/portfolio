About

I’m a software developer transitioning into data science and machine learning. My focus spans applied ML for systems (IoT/SDN), text (NLP), and vision (UAV/traffic).

I value clean repositories, reproducibility, and well-scoped tests. Code should be easy to run, documented, and paired with short demos that prove utility quickly.

I prefer practical baselines before complex models, and I maintain simple pipelines that scale. I’m currently consolidating projects that reflect these principles.

FAQ

How do you keep repositories clean?
Minimal dependencies, clear READMEs, pinned versions, and small setup scripts. Reproducibility is a priority.
Do you include tests?
Yes—lightweight checks for core components and pipelines so changes remain safe.
What do “short demos” mean?
Small scripts/notebooks that run in minutes and show practical impact quickly.
